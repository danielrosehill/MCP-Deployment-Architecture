# v2 — 2026-04-28

Two changes versus [v1](v1-2026-04-26.md): the workstation-local aggregator goes away, and the lightweight HTTP staging service is replaced by MinIO on the LAN server.

## What changed

### File transfer — MinIO replaces the HTTP staging service

The lightweight HTTP staging microservice is being superseded by **MinIO** running on the LAN server. MinIO provides:

- **S3-compatible API** access from inside and outside the LAN
- **Presigned URL** flows for clients without filesystem access (phones, browsers, remote agents)
- **Lifecycle / retention policies** so artifacts staged temporarily are pruned automatically rather than accumulating storage indefinitely
- A more robust on-ramp for binary payloads that any MCP server with object-store awareness can consume

This generalises the staging layer from a bespoke HTTP shim into a standard, well-understood service surface. External API access (without first being on the LAN) becomes practical.

### Aggregation — workstation-local aggregator removed

The localhost MCP aggregator is being **removed**. The workstation now makes **direct connections** to the small number of edge-case servers that genuinely need to run locally (the same ones described in v1: tools requiring local device access, tools without remote-file-transfer support, tools too cumbersome to configure remotely).

The LAN VM aggregator remains in place and continues to carry the bulk of the catalog.

In practice the localhost aggregator added a layer that paid less than its cost given how few servers actually need to live on the workstation. A direct-connection pattern is simpler to reason about and to debug for that small residual set.

## Net effect

- **MetaMCP / MCP Jungle on the LAN VM** — the bulk of aggregation, unchanged from v1.
- **Workstation** — no aggregator; direct point-to-point connections to the small set of servers that must run locally.
- **MinIO on the LAN VM** — file staging as a first-class service with retention.
- **LAN device tier and remote-access tier (Cloudflare Tunnels, Tailscale, OpenClaw gateway)** — unchanged from v1.
